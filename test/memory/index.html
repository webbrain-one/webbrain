WebBrain Memory Tutorial

A guided local page for testing explicit memory, optional auto-learning, form-derived memory, replaying saved preferences into a later form, and Profile auto-fill. Use fake data only.

Setup 1. Slash remember 2. Form memory 3. Auto-learn 4. Replay memory 5. Profile auto-fill

Setup

Serve this folder with a static server, then open this page in a normal tab: 

python3 -m http.server 8765 -d test/memory
http://127.0.0.1:8765/

WebBrain settings

  1. Open WebBrain Settings, then open the Memory tab.
  2. Turn on Memory.
  3. Turn on Auto-learn memories for steps 2 and 3.
  4. Turn on Learn from completed forms for step 2.
  5. Clear old memories before a clean run, or export them first.

What counts as a pass

  • /memory --add saves immediately and appears in /memory.
  • Auto-learning runs after the assistant response, so it can take a few seconds.
  • Form-derived learning saves durable preferences, not raw form field dumps.
  • The final forms are filled from saved memory and Profile text without re-asking.
Do not enter real secrets, real passwords, API keys, bank details, or private account data. This page is intentionally local and static, but WebBrain memory is sent to your configured LLM provider when injected.
1

Slash remember command

Save a known preference without waiting for the extractor.

Paste in WebBrain 
/memory --add Prefer memory tutorial answers in concise bullets, use USD for budgets, and use America/New_York when scheduling.

Then verify

  1. Run /memory.
  2. Confirm a memory appears with the text above or an equivalent normalized version.
  3. Open Settings, Memory tab, and confirm the same record appears in the editable list.

Form-derived memory with opt-in enabled

Fill out the intake form yourself, then have WebBrain read it, ask a durable-preference clarification, and submit it.

Human action

  1. Fill this form with the sample values or your own fake preferences.
  2. Ask WebBrain to process the form using the prompt below.
  3. If WebBrain asks what to remember, answer with a durable preference sentence.
  4. If WebBrain asks to submit, answer submit.
  5. Wait, then run /memory.
Paste in WebBrain 
Submit Step 2 form.
Suggested clarification answer 
Remember that for future travel and event forms I prefer window seats, vegetarian meals, email receipts, quiet hotels, and one-page summaries.
Expected memory: a durable preference or workflow hint like "Prefer window seats, vegetarian meals, email receipts, quiet hotels, and one-page summaries for travel and event forms."

3

Automatic learning after a normal chat

Ask for a stable preference in chat and wait for the background extractor.

Paste in WebBrain 
For future work with me, remember that I prefer replies with exactly two sections: Decision and Next step. Please acknowledge this preference.

Verify after the turn

  1. Wait 10 to 30 seconds after the assistant finishes.
  2. Run /memory, or refresh Settings, Memory tab.
  3. Expect a new memory about the two-section reply format.
