根据耐心、隐私需求与预算选择方案。

WebBrain 支持内置云服务、直接云 API、模型路由和本地 OpenAI 兼容服务器。你只需要一个活动提供商。

快速选择

步骤最少

WebBrain Cloud

内置且默认启用，包含每日免费额度，无需模型服务器或 API 密钥。

隐私与离线能力

本地模型

使用 llama.cpp、Ollama、LM Studio、Jan、vLLM、SGLang 或 LocalAI。

直接使用厂商

云 API

为 OpenAI、Anthropic、Gemini、Mistral、DeepSeek、xAI、Groq 等提供商填写自己的密钥。

模型目录丰富

模型路由

使用一个账户切换许多模型；适合经常试验的用户。

配置步骤

  1. 找到提供商。使用“本地”“云端”“路由”或搜索框。
  2. 展开卡片并填写值。本地服务器通常需要地址和模型名；云提供商还需要 API 密钥。
  3. 保存。把当前字段写入浏览器。
  4. 测试连接。先修复地址、密钥、模型或服务器错误。
  5. 设为活动。只有测试成功后再切换。保存并不等于已经启用。

字段含义

字段含义建议
服务器 / Base URLOpenAI 兼容根端点或提供商 API 根地址。使用本地应用或提供商给出的精确值，不要粘贴聊天网页地址。
API 密钥发送给提供商用于认证的凭据。本地服务器通常不需要，除非你主动启用了认证。
模型服务器要求的模型标识符。大小写、前缀和拼写必须准确。
上下文窗口一次请求容纳提示、历史、网页内容和工具模式的容量。可靠本地运行建议至少 16k；8k 可配合 Compact；4k 太小。
支持视觉当前模型是否接受图像。只有确认所服务的精确模型支持图像时才开启。
提示层级WebBrain 暴露的提示长度和常规浏览器工具范围。本地模型从 Mid 开始；小上下文用 Compact，能力强的模型用 Full。

本地模型的额外环节

服务器常见 Base URLAPI 密钥
llama.cpphttp://localhost:8080通常不需要
Ollamahttp://localhost:11434/v1不需要
LM Studiohttp://localhost:1234/v1不需要
Janhttp://localhost:1337/v1可选
vLLMhttp://localhost:8000/v1可选
SGLanghttp://localhost:30000/v1可选
终端可用，不代表扩展一定可访问

连接被拒绝通常是服务器未启动或端口错误；CORS/Origin 错误需要允许浏览器扩展来源。Ollama 常用：OLLAMA_ORIGINS="chrome-extension://*,moz-extension://*" ollama serve。

提示层级

Compact最短提示与最小工具面，适合约 8k 上下文的小型本地模型。
Mid包含常见浏览器任务、iframe、下载、计划任务和表单验证。推荐的本地起点。
Full增加 hover、拖放、框架与 Shadow DOM 等高级回退工具。

云提供商固定使用 Full；本地提供商默认 Mid；Compact 提供商不能使用 Dev 模式。

测试连接失败时

  1. Connection refused：启动本地服务器并确认端口。
  2. CORS/Origin：在服务器中允许 WebBrain 扩展来源。
  3. Unauthorized：确认密钥与 Base URL 属于同一提供商。
  4. Model not found：复制精确模型 ID，或使用 Load models。
  5. 超时：为大型 CPU 本地模型提高常规请求超时。
  6. 几轮后失败：确认真实上下文窗口，降低提示层级或扩大模型上下文。
